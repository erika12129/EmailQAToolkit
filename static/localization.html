How to Use Localization QA

  1. Upload your localized email HTML file and its corresponding requirements JSON.
  2. Optionally check "Compare with parent email" to enable comparison with the parent (en_US) version.
  3. If comparing with parent, upload the parent email HTML and requirements JSON files.
  4. Click "Run Localization QA" to process the validation and comparison.

What This Tool Checks:

  • Localized Requirements: Validates the localized email against its specific requirements (subject, preheader, sender, UTM parameters).
  • Parent Requirements (Optional): Validates if the localized email would also pass the parent requirements.
  • Link Structure Comparison (Optional): Checks if the localized email links follow the same structure as the parent email links, accounting for locale-specific differences like domain changes (e.g., .com vs .com.mx) and language path changes (e.g., /en/ vs /es/).
